# Reverse Brainstorming — Field Manual

Reverse brainstorming exploits a cognitive trick: people find it easier (and more
honest) to imagine how to *break* something than how to *fix* it. You generate the
failure list, then flip it.

## Procedure

### Step 1 — Reverse the question
Turn the goal into its opposite:
- Framing (Officer 1): "How could we make this problem far worse / guarantee it
  happens?"
- Solution (Officer 3): "How could we guarantee this solution fails / cause this
  failure?"

### Step 2 — Brainstorm the negatives (diverge)
Apply the brainstorming rules (defer judgment, quantity, wild ideas, build).
Generate 10–20 concrete mechanisms that would *cause/worsen* the problem. Each
must be a real mechanism, not "do it badly".

### Step 3 — RE-INVERT (mandatory)
For **each** negative, write its reversal:
- A "worsen" mechanism → the preventive/corrective measure that neutralizes it.
- Pair them explicitly (negative ↔ positive). This step is non-optional.

### Step 4 — Harvest insight
- For Officer 1: what do the worst-case mechanisms reveal about what the problem
  *really is* / its key risk dimensions? Sharpen the framing.
- For Officer 3: collect the reversed measures as candidate solutions.

### Step 5 — Flag bold + facts
Mark the 2–3 most revealing pairs. Cite or flag any factual claim.

## Output format

```
CIBLE : <framing (O1) | solution (O3)> de <…>
QUESTION INVERSÉE : <"comment aggraver / faire échouer …">

| # | Mécanisme négatif (aggraver/échouer) | Renversement (mesure/insight)        | Note |
|---|--------------------------------------|--------------------------------------|------|
| 1 | …                                    | …                                    | candidat / source / à vérifier |
| 2 | …                                    | …                                    |      |
|…  | (viser 10–20)                        |                                      |      |

INSIGHT (O1: cadrage affiné / O3: solutions candidates) : <synthesis>
PAIRES À CREUSER : <2–3 ids>
SOURCES : <numbered list>   |   À VÉRIFIER : <list>
```

## Guardrails
- BOTH flips are mandatory — never deliver only the negatives.
- Inversions must be concrete mechanisms, not "do X badly".
- Candidates, not facts; cite or flag factual claims.
- Generation only — no decision. Mirror the user's language.
