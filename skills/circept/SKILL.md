# CIRCEPT — Field Manual

CIRCEPT explores a concept *in the round*: every concept has neighbors (what it
evokes) and opposites (its polarities). Mapping both enriches meaning and unlocks
fresh angles that a linear definition misses.

## When to use
- A problem/concept feels vague, one-dimensional, or "stuck".
- Before framing (O1) or when generating solution angles (O3).

## Procedure

### Step 1 — Set the central concept
Name the single concept at the center (e.g. the core noun of the problem).

### Step 2 — Map associations (the circle)
List what the concept evokes: attributes, near-synonyms, contexts, sensations,
metaphors. Go around the circle; include 2–3 *distant/unexpected* associations to
break fixation.

### Step 3 — Map opposites (the polarities)
For the concept and its key associations, give genuine opposites — not lazy
negations ("not X") but real polar concepts (e.g. speed ↔ endurance, not "slow").

### Step 4 — Read the tensions
The interesting insight usually sits on an association↔opposite tension. Note 2–3.

### Step 5 — Reframe
Turn the richest tensions into:
- O1: alternative problem framings.
- O3: candidate solution angles.

## Output format

```
CONCEPT CENTRAL : <concept>   (pour Officier <n>)

ASSOCIATIONS (cercle) :
   proches : …, …, …
   lointaines/inattendues : …, …
OPPOSÉS (polarités) :
   <concept/assoc> ↔ <opposé réel>
   …  (vrais opposés, pas "non-X")

TENSIONS CLÉS : <2–3 association ↔ opposé qui éclairent>
REFRAMES / ANGLES : <2–3 — cadrages (O1) ou pistes solution (O3)>
SOURCES : <faits cités>   |   À VÉRIFIER : <list>
```

## Guardrails
- Map BOTH associations AND real opposites — never associations alone.
- Opposites must be genuine polarities, not negations.
- Associations are candidates, not facts; cite/flag any factual claim.
- Enrich/reframe only — no decision. Mirror the user's language.
