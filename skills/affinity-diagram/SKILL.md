# Affinity Diagram (KJ) — Field Manual

The affinity diagram organizes many items by *natural relationship* instead of
logical/predefined categories. Themes emerge from the data, then get named.

## When to use
- After a brainstorm, interviews, a survey, or a pile of symptoms/data.
- When the problem feels "messy" and needs structure before framing.

## Procedure

### Step 1 — Collect the items
List all items, one idea per line, in the participant's own words where possible.
Mark any factual item with its source (or `à vérifier`).

### Step 2 — Group silently by affinity (bottom-up)
Cluster items that "feel related" — do NOT use predefined buckets. Aim for 4–8
groups. An item may be split if it carries two ideas. Items that fit nowhere go to
a "À clarifier" group (never drop them).

### Step 3 — Name each group (header)
Write a short header that captures the *essence* of the group (a phrase, not a
single vague word). A good header could stand alone as a finding.

### Step 4 — (Optional) relate the groups
Note any obvious super-groups or links between themes.

### Step 5 — Read the structure
One line: what does this emergent structure suggest about the problem?

## Output format

```
ITEMS : <n total>

▸ <Header thème 1>
   - item …            [verbatim | source/url | à vérifier]
   - item …
▸ <Header thème 2>
   - item …
▸ … (4–8 thèmes)
▸ À CLARIFIER
   - item …            [orphan / placement incertain]

LECTURE : <one line — what the structure reveals>
SOURCES : <numbered list of factual items>
```

## Guardrails
- Bottom-up: never force items into predefined categories.
- Place every item; orphans go to "À clarifier", flagged — never silently dropped.
- Facts sourced; grouping is interpretation, labeled as such.
- Organize only — no cause diagnosis, no decision. Mirror the user's language.
